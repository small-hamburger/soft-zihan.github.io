# 00 走进终端 (Introduction)

欢迎来到 Linux 的世界！在开始学习命令之前，我们需要先了解几个核心概念。

## 1. 什么是终端 (Terminal) 与 Shell？
- **终端 (Terminal)**：是你输入命令、查看输出的那个“窗口”。比如 Ubuntu 上的 `Gnome Terminal`，KDE 上的 `Konsole`。
- **Shell**：是窗口背后的“解析器”，它负责解析你输入的指令并传达给操作系统内核。
  - **Bash**：绝大多数 Linux 发行版的默认 Shell。
  - **Zsh**：现代化的 Shell，功能更强（本教程中的很多“Pro Tip”配合 Zsh 效果最佳）。

## 2. 你在屏幕上看到了什么？(理解提示符)
当你打开终端，你会看到类似这样的一行字：
`han@ubuntu:~/Documents$ `
- **han**：当前登录的用户名。
- **ubuntu**：计算机的名字 (Hostname)。
- **~/Documents**：你当前所在的文件夹路径 (`~` 代表家目录)。
- **$**：普通用户提示符 (如果是 **#** 则代表你是 root 管理员)。

## 3. 核心准则：Linux 是区分大小写的
在 Windows 中，`File.txt` 和 `file.txt` 是同一个文件；但在 Linux 中，它们是**两个完全不同**的文件。命令也是一样，`ls` 有效，但 `LS` 会报错。

## 4. 授人以渔：如何获取帮助？
遇到不认识的命令？不要只靠百度，学会使用 Linux 自带的“说明书”：
- **`命令 --help`**：查看简要的参数说明 (最常用)。
- **`man 命令`**：查看详尽的开发手册 (Manual)。按 `q` 退出。
> 💡 **Pro Tip**: 安装 **`tldr`**。它是 `man` 的简化版，只给你看最常用的例子，非常适合新手。

## 5. 如何安装工具？
根据你的系统，使用对应的包管理器：
- **Ubuntu/Debian**: `sudo apt update && sudo apt install 工具名`
- **CentOS/RHEL/Fedora**: `sudo dnf install 工具名` (旧版使用 `yum`)
- **Arch Linux**: `sudo pacman -S 工具名`

### 🚀 核心概念：什么是 PATH (环境变量)？
为什么你输入 `ls` 就能执行，而输入自己写的脚本却提示找不到？
- **PATH** 是一个“搜索清单”（文件夹列表）。
- 当你输入命令时，系统会按顺序去这些文件夹里找对应的程序。
- **查看清单**：输入 `echo $PATH`。
- **理解**：PATH 就像是系统的“拨号快捷键”，在清单里的程序可以随时随地“拨号”运行。
